# AI Manager Skills - Module to Concept Mapping

## The Six Core Managerial Concepts

| # | Concept | Definition |
|---|---------|------------|
| 1 | **Context Assembly** | Gathering and presenting relevant background information to enable effective AI collaboration |
| 2 | **Quality Judgment** | Critically evaluating AI outputs for accuracy, completeness, and fitness for purpose |
| 3 | **Task Decomposition** | Breaking complex problems into smaller, AI-appropriate components |
| 4 | **Iterative Refinement** | Progressively improving outputs through cycles of feedback and revision |
| 5 | **Workflow Integration** | Embedding AI collaboration into existing work processes sustainably |
| 6 | **Frontier Recognition** | Understanding the current boundaries of AI capability |

## PHASE 3: WORKFLOW (Weeks 7-10)

### Week 7: Task Decomposer
**Primary Concept:** Task Decomposition

| Aspect | Description |
|--------|-------------|
| **Focus** | Categorizing tasks by AI-appropriateness and mapping dependencies |
| **Key Activities** | Classify tasks as AI-Optimal, Collaborative, or Human-Primary; sequence by dependencies |
| **Learning Outcome** | Good decomposition is 90% of effective AI delegation |
| **Tool Output** | Task category maps, dependency diagrams, parallel track identification |

**The Three Categories:**
1. **AI-Optimal:** Well-defined I/O, pattern-based, no institutional knowledge needed
2. **Collaborative:** Requires judgment, benefits from context + AI capability
3. **Human-Primary:** Requires authority, credentials, or institutional knowledge

**Supporting Concepts:**
- Quality Judgment (knowing which tasks AI does well)
- Workflow Integration (decomposition enables workflow design)

### Week 9: Iteration Passes
**Primary Concept:** Iterative Refinement

| Aspect | Description |
|--------|-------------|
| **Focus** | Structured iteration using the 70-85-95 framework |
| **Key Activities** | Label passes explicitly; use transition templates; track iterations-to-done |
| **Learning Outcome** | Labeled passes reduce total iterations and prevent scope creep |
| **Tool Output** | Pass-labeled feedback logs, transition templates, iteration statistics |

**The Three Passes:**
| Pass | Target | Focus | Key Question |
|------|--------|-------|--------------|
| 1 | 70% | Structure & Approach | Right problem, right way? |
| 2 | 85% | Robustness | What will break in practice? |
| 3 | 95% | Production-Ready | Will this work for its audience? |

**Supporting Concepts:**
- Quality Judgment (each pass requires judgment)
- Task Decomposition (passes are discrete phases)

### Week 10: Status Reporter
**Primary Concept:** Workflow Integration

| Aspect | Description |
|--------|-------------|
| **Focus** | Embedding AI into recurring workflows with measurable outcomes |
| **Key Activities** | Design workflow templates (inputs, steps, prompts, checks); track time savings; generate status reports |

**Supporting Concepts:**

- Iterative Refinement (workflows improve over time)

## PHASE 4: ADVANCED (Weeks 11-12)

### Week 11: Frontier Mapper
**Primary Concept:** Frontier Recognition

| Aspect | Description |
|--------|-------------|
| **Focus** | Mapping where AI reliability drops off in your domain |
| **Key Activities** | Document reliability zones; probe frontier areas with verification questions; log frontier encounters |
| **Learning Outcome** | Frontier awareness prevents expensive mistakes in unreliable zones |
| **Tool Output** | Domain reliability maps, encounter logs, confidence assessments |

**The Four Reliability Territories:**
| Territory | Description | Approach |
|-----------|-------------|----------|
| **Reliable** | Well-documented, established patterns | Trust with basic verification |
| **Frontier Zone** | Newer developments, complex reasoning | Trust but verify carefully |
| **Beyond Frontier** | Cutting-edge, organization-specific | Use as starting point only |
| **Outside Scope** | What AI fundamentally can't know | Don't rely on |

**Supporting Concepts:**
- Quality Judgment (frontier awareness informs verification needs)
- Trust calibration from Week 5 feeds into frontier mapping
